# 📚 项目文档目录

## 目录结构说明

### 01-需求文档/
- **内容**: 产品需求文档、用户故事、功能规格等
- **文件类型**: .md, .docx, .pdf
- **维护**: 产品经理、需求分析师

### 02-设计文档/
- **内容**: 技术设计、架构设计、数据库设计等
- **文件类型**: .md, .drawio, .png
- **维护**: 架构师、技术经理

### 03-计划文档/
- **内容**: 项目计划、进度跟踪、里程碑等
- **文件类型**: .md, .xlsx, .csv
- **维护**: 项目经理、团队负责人

### 04-测试文档/
- **内容**: 测试用例、测试报告、质量评估等
- **文件类型**: .md, .xlsx, .csv
- **维护**: 测试工程师、质量保证

### 05-团队文档/
- **内容**: 团队分工、角色职责、协作流程等
- **文件类型**: .md, .docx
- **维护**: 项目经理、团队负责人

### 06-会议记录/
- **内容**: 会议记录、决策记录、行动计划等
- **文件类型**: .md, .docx
- **维护**: 会议记录员、项目经理

## 文档管理规范

### 命名规范
- 使用英文或拼音命名
- 包含日期和版本信息
- 避免特殊字符和空格
- 示例: `PRD_v1.0_20260309.md`

### 版本控制
- 重要文档使用版本号
- 变更记录在文档中
- 旧版本保留在历史目录
- 使用Git进行版本管理

### 更新流程
1. 创建或修改文档
2. 添加变更记录
3. 提交Git版本
4. 通知相关方
