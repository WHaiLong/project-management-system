### 失败情况及处理

#### 情况1：SSH权限被拒绝
- **可能原因**: SSH公钥未添加到码云账户
- **解决方案**:
  1. 生成SSH密钥对（如果还没有）
  2. 将公钥添加到码云账户设置
  3. 测试SSH连接：`ssh -T git@gitee.com`
  4. 重新执行推送

#### 情况2：仓库不存在
- **可能原因**: 仓库路径错误或未创建
- **解决方案**:
  1. 登录码云确认仓库是否存在
  2. 如果不存在，创建新仓库
  3. 确认仓库路径正确
  4. 更新远程URL或重新添加

#### 情况3：网络连接问题
- **可能原因**: 网络问题、DNS解析、代理设置
- **解决方案**:
  1. 检查网络连接
  2. 测试域名解析：`ping gitee.com`
  3. 检查代理设置
  4. 尝试HTTPS方案

#### 情况4：HTTPS认证失败
- **可能原因**: 账号密码错误、无访问权限
- **解决方案**:
  1. 确认码云账号密码正确
  2. 使用访问令牌代替密码
  3. 检查仓库访问权限
  4. 联系仓库管理员

## 🔧 技术方案

### SSH方案（首选）
```bash
# 1. 生成SSH密钥（如果还没有）
ssh-keygen -t rsa -C "your_email@example.com"

# 2. 查看公钥
cat ~/.ssh/id_rsa.pub

# 3. 将公钥添加到码云
#    登录码云 → 设置 → SSH公钥 → 添加公钥

# 4. 测试连接
ssh -T git@gitee.com

# 5. 配置远程
git remote add gitee git@gitee.com:WHaiLong/project-management-system.git

# 6. 推送代码
git push gitee main --force
```

### HTTPS方案（备用）
```bash
# 1. 配置HTTPS远程
git remote add gitee-https https://gitee.com/WHaiLong/project-management-system.git

# 2. 推送代码（会提示输入凭据）
git push gitee-https main --force

# 3. 使用凭据助手（避免重复输入）
git config --global credential.helper store
```

### 访问令牌方案（最安全）
```bash
# 1. 获取码云访问令牌
#    登录码云 → 设置 → 私人令牌 → 生成新令牌

# 2. 使用令牌推送
TOKEN="你的访问令牌"
git remote set-url gitee https://oauth2:${TOKEN}@gitee.com/WHaiLong/project-management-system.git
git push gitee main --force
```

### 步骤2：验证代码一致性
```bash
# 比较本地和远程提交
LOCAL_HASH=$(git log -1 --format="%H")
GITEE_HASH=$(git ls-remote gitee main | awk '{print $1}')

echo "本地: $LOCAL_HASH"
echo "码云: $GITEE_HASH"

if [ "$LOCAL_HASH" = "$GITEE_HASH" ]; then
    echo "✅ 代码一致"
else
    echo "❌ 代码不一致"
fi
```
