# 🔧 脚本工具目录

## 目录结构说明

### 01-环境配置/
- **内容**: 环境安装、配置、验证脚本
- **文件类型**: .sh, .bat, .ps1
- **用途**: 开发环境、测试环境、生产环境配置
- **示例**: 安装MySQL、配置GitLab、环境检查

### 02-部署脚本/
- **内容**: 应用部署、发布、回滚脚本
- **文件类型**: .sh, .yaml, .json
- **用途**: 自动化部署、持续集成、发布管理
- **示例**: Docker部署、Kubernetes配置、发布流水线

### 03-工具脚本/
- **内容**: 开发工具、辅助工具、自动化脚本
- **文件类型**: .sh, .py, .js
- **用途**: 代码生成、数据迁移、批量处理
- **示例**: 代码统计、数据备份、文件整理

### 04-测试脚本/
- **内容**: 测试执行、验证、报告脚本
- **文件类型**: .sh, .py, .js
- **用途**: 自动化测试、性能测试、安全测试
- **示例**: API测试、数据库测试、压力测试

## 脚本使用规范

### 命名规范
- 使用英文描述性名称
- 包含主要功能说明
- 使用小写字母和下划线
- 示例: `setup_development_env.sh`

### 脚本头信息
每个脚本应包含：
```bash
#!/bin/bash
# 脚本名称: setup_development_env.sh
# 功能描述: 设置开发环境
# 作者: 吴工
# 创建日期: 2026-03-09
# 最后修改: 2026-03-09
# 版本: 1.0
```

### 参数说明
- 使用标准参数格式
- 提供帮助信息
- 验证参数有效性
- 提供默认值

### 错误处理
- 检查命令执行结果
- 提供有意义的错误信息
- 清理临时资源
- 记录错误日志

## 常用脚本

### 环境配置
- `setup_mysql.sh` - MySQL安装和配置
- `check_environment.sh` - 环境检查
- `configure_gitlab.sh` - GitLab配置

### 部署脚本
- `deploy_backend.sh` - 后端部署
- `deploy_frontend.sh` - 前端部署
- `rollback_deployment.sh` - 回滚部署

### 工具脚本
- `code_statistics.sh` - 代码统计
- `organize_project.sh` - 项目整理
- `backup_data.sh` - 数据备份

### 测试脚本
- `run_api_tests.sh` - API测试
- `test_database.sh` - 数据库测试
- `performance_test.sh` - 性能测试

## 安全注意事项

### 权限管理
- 敏感脚本设置适当权限
- 避免硬编码密码和密钥
- 使用环境变量或配置文件
- 定期审查脚本安全性

### 执行安全
- 在生产环境谨慎执行
- 先测试再执行
- 备份重要数据
- 监控脚本执行
